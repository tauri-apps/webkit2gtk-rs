fn get_favicon<P: IsA<Cancellable>, Q: FnOnce(Result<Surface, Error>) + Send + 'static>(
    &self,
    page_uri: &str,
    cancellable: Option<&P>,

)[src]

Asynchronously obtains a cairo::Surface of the favicon for the given page URI. It returns the cached icon if it's in the database asynchronously waiting for the icon to be read from the database.

This is an asynchronous method. When the operation is finished, callback will be invoked. You can then call FaviconDatabaseExt::get_favicon_finish to get the result of the operation.

You must call WebContextExt::set_favicon_database_directory for the WebContext associated with this FaviconDatabase before attempting to use this function; otherwise, FaviconDatabaseExt::get_favicon_finish will return FaviconDatabaseError::NotInitialized.

page_uri

URI of the page for which we want to retrieve the favicon

cancellable

A gio::Cancellable or None.

callback

A GAsyncReadyCallback to call when the request is satisfied or None if you don't care about the result.

user_data

The data to pass to callback.

fn connect_favicon_changed<F: Fn(&Self, &str, &str) + 'static>(
    &self,
    f: F
) -> SignalHandlerId[src]

This signal is emitted when the favicon URI of page_uri has been changed to favicon_uri in the database. You can connect to this signal and call FaviconDatabaseExt::get_favicon to get the favicon. If you are interested in the favicon of a WebView it's easier to use the WebView:favicon property. See WebViewExt::get_favicon for more details.
